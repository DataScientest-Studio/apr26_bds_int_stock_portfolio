Glossary

Asset / the 7 files
One ticker folder under Assets/<T>/: executed notebook, three OHLCV parquets, Optuna best params JSON, standalone strategy, and OOS README.
Feature namespaces
ID ranges define the model input lanes: 01-99 for 1h, 101-199 for 1d, 201-299 for 1w, and 901-999 for multi-timeframe features.
Candidate bar
A 1h bar with sufficient history and nonzero log_return_5. Its side is the sign of that momentum value.
Triple Barrier Y
The binary label from a symmetric ATR barrier around the next-open entry, with target, stop, and time-barrier outcomes.
Optuna / XGB
Optuna tunes the XGBoost binary classifier on Train-only purged walk-forward CV. XGB predicts p(Y=1).
Kelly sizing
Per-trade fraction f = clip(lambda * (2p - 1), 0, KELLY_CAP), calibrated on Train out-of-fold growth.
Dashboard
Static page reading Plan/data/dashboard.json, exported from Project/Structure/oos_metrics.db.
ML Basket Simulator (client app)
Tier 1 — the Streamlit demo (make app, port 8501). Each picked ticker is a $1000 entry on the first OOS day; the basket outcome comes read-only from oos_metrics.db. Nothing is retrained at runtime.
Documentation tiers
Tier 1 — Client app (the simulator). Tier 2 — Backend explanation (these Plan/ pages). Tier 3 — Replication blueprint (procedure_lego.html + the "Kontrakt replikacji" blocks in Project/endproduct/Layers_Short_SOT.md): how to rebuild an analogous app from accumulating data-processing modules.
