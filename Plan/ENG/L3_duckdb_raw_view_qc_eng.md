# L3 · DuckDB + QC (summary)

- From the zips we load a single DuckDB database (file `*.duckdb`).
- The database holds the table `raw_ohlcv_1h`.
  - key columns: `symbol VARCHAR`, `ts TIMESTAMP`
  - price columns: `open/high/low/close BIGINT ×10000`
  - volume column: `volume BIGINT`
  - row count: 8 841 820
  - symbol count: 503 (ticker list: file `config/universe.txt`)
- USD exists only in the view `VIEW ohlcv_1h` (= prices `/10000.0`).
  - this view is `price_view = raw_usd_view` from the input contract (`build_contract_eng.md` §Input)
  - the view does not copy data (zero storage duplication)
- The `_meta` table holds the schema version, source and counters.
- Every load passes 11 gates QC-01…QC-11:
  - QC-01: high ≥ low on every candle
  - QC-02: high ≥ max(o,c) · low ≤ min(o,c)
  - QC-03: no duplicate `(symbol, ts)`
  - QC-04: zero NULL in o/h/l/c/v
  - QC-05: all prices > 0
  - QC-06: volume ≥ 0
  - QC-07: universe 503/503 symbols
  - QC-08: candles/day ∈ [5,9]
  - QC-09: ts within session 09:00–16:00 ET
  - QC-10: ts strictly increasing per symbol
  - QC-11: date range and counters match `_meta`
- a fail of any gate = the load is not published
- Per-symbol upsert works as DELETE-then-INSERT.
