A truly rotatable three-dimensional visualization of the ML pipeline for 503 S&P 500 companies in HTML Canvas, yaw and pitch orbit by mouse drag, gentle perspective, depth-sorted painting. Twelve levels from the bottom: an Alpaca SIP 1h source for the S&P 500 (download → 510 ZIPs; quality-filtered to 503 downstream), a LEAN ZIP store times 510 with prices times 10000, DuckDB with a raw table and an ohlcv_1h (USD) view plus a discreet row of quality-gate dots QC-01…QC-11, an atomic snapshot to a per-asset ticker folder whose run notebook (<TICKER>__L4_to_L12.ipynb) creates the OHLCV parquet from the snapshot and drives the asset through to L12, a warm-up train OOS time split with purge and embargo, a trend-line setup detector that finds L_trend, L_opp, entry_candle and R0 using only candles up to t0, trendline features X with a triple barrier label Y, a data-quality gate over stores and transforms with a dashboard before training, an Optuna hyperparameter search that produces OPTUNAs_XGB_HPOs_best_params.json, a final XGBoost training that serializes each champion model as a base64-encoded artifact into a per-asset strategy file (×503 independent files, one model per asset), a SQLite asset-metrics database (l11_asset_metrics.sqlite) holding a 503-asset metric matrix and a distribution report, and a per-asset ticker folder (×503) holding five files — the <TICKER>__L4_to_L12.ipynb runner, the OHLCV parquet, the OPTUNAs_XGB_HPOs_best_params.json, the strategy_<TICKER>.py and a human-readable <TICKER>_README.md report (this asset's params, OOS verdict summary and data info). Horizontal dashed separators between layers and editable note-cards for L-to-L transitions. View modes 1 to 9, 0 and L, pan, zoom, tooltips, side panel, label editor, PNG export.

  1. Alpaca SIP · 1h — 510 downloaded ZIP tickers; 503 quality-filtered ML universe downstream (L3), session 09:00–16:00 ET (~7 candles/day), 2016 → today.
  2. LEAN ZIP: 510 zips (1 per ticker), CSV with prices in deci-cents ×10000, 139 MB.
  3. DuckDB: raw table (BIGINT ×10000) + VIEW ohlcv_1h (USD, /10000.0); every load is gated by QC-01…QC-11.
  4. Snapshot → parquet OHLCV per ticker (×503), written by the per-asset <TICKER>__L4_to_L12.ipynb runner from the DuckDB snapshot — features are computed only by the transformer (L7).
  5. Time split: warm-up → train → OOS; purge H=24 + embargo, because TB labels overlap the boundaries.
  6. Trend-line setup detector — fits L_trend, L_opp, touchpoints, t0, R0, direction using only candles ≤ t0 (zero look-ahead).
  7. Trendline features = 7 geometric X + closed_through_line audit (8 transformer columns); FEATURE_MANIFEST = 8 X (7 geom + direction) + label Y (triple barrier: TP=1 · SL=0 · time=0, close-based, SL=L_opp(t) moving, barriers from R0).
  8. Data-quality gate (dashboard): parities zip→DuckDB→parquet→Output B, gaps, zero-values, NaN/Inf, split assertion — FAIL blocks training.
  9. Optuna hyperparameter search on Train (200 trials, TPE + MedianPruner, purged walk-forward CV) → OPTUNAs_XGB_HPOs_best_params.json.
  10. XGB final training per asset on OPTUNAs_XGB_HPOs_best_params.json; champion → base64 MODEL_B64 embedded inside strategy_<TICKER>.py (decoded via import base64 at import) + selfcheck → ×503 independent files (1 model/hash/window per asset).
  11. OOS single run → l11_asset_metrics.sqlite: 503-asset metric matrix (PF · MDD · TIM · WR) + distribution report.
  12. Endproduct: per-asset ticker folder <TICKER>/ = <TICKER>__L4_to_L12.ipynb (runner) + <TICKER>_ohlcv_1h.parquet + OPTUNAs_XGB_HPOs_best_params.json + strategy_<TICKER>.py + <TICKER>_README.md (×503 folders).
