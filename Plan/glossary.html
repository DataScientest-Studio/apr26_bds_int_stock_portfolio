Glossary

The vocabulary of the minimal pipeline. ← back

Structure
Layer (1.6 … 4.2)
A single stage of the pipeline. Before them: the OHLCV was acquired upstream from Alpaca and held in QuantConnect/LEAN zip (1.3–1.4, provenance only), exported to data/seed, which build_db.py (1.5) loads into the shared liora.duckdb input. The runner notebook then executes layers 1.6 … 4.2 in order; together they take one ticker from clean OHLCV to an out-of-sample report.
Asset / the 7 files
One ticker’s deliverable folder Assets/<T>/: the executed notebook, three OHLCV parquets (1h/1d/1w), the Optuna best-params JSON, the standalone strategy .py, and the OOS README.
Data acquisition (1.3–1.4) — Alpaca · QuantConnect/LEAN
Upstream provenance — not code in this repo. The raw 1h OHLCV was downloaded from the Alpaca Market Data API (feed=sip), held in QuantConnect/LEAN one-zip-per-ticker archive format, then exported once to data/seed/*_ohlcv_1h.parquet. This self-contained repo ships those parquets and never calls the API live; the visualization shows this at the very bottom.
Data source / input store (1.5) / liora.duckdb
The single input store: a DuckDB table bars_1h(ticker, timestamp, open, high, low, close, volume), built from the exported data/seed/*_ohlcv_1h.parquet by build_db.py (make build-db). This is the build_db step (1.5), just above the upstream acquisition (1.3–1.4).
Data & timeframes
OHLCV
Open / High / Low / Close / Volume — the raw candle. Timestamps are interval-open, UTC.
1h / 1d / 1w
The source 1-hour bars, plus deterministic roll-ups to ET calendar day (1d) and ISO week (1w). 1d/1w are pure-OHLCV, used only as causal context.
Warmup / Train / OOS
The three time windows. Warmup primes indicators; the model is tuned and trained on Train; OOS is a single untouched out-of-sample test.
Purge & embargo
Gaps that stop a label’s outcome window from overlapping the Train→OOS boundary, so there is no look-ahead leakage.
Signal & features
Setup (trend-line detector)
A causal pattern: a fitted trend line touched MIN_TOUCHES times, yielding a candidate long/short entry. DET-09 rejects degenerate / zero-ATR setups.
FT1–FT8
The eight standard trendline features (distance to line, Risk-Box height, bar return, body/range, volume z-score, touch count, direction, …) computed per setup.
Output-B
The per-setup feature matrix X plus the label Y and audit columns — the table the model trains on.
Triple-barrier (TB_v1.2) / label Y
The outcome label: over the next H bars, did price hit the target, the safety line, or neither? Defines win/loss.
Feature manifest
The resolved, per-asset list of X columns (the 1h set by default; some assets add 1d/1w context).
Training & OOS
Optuna / TPE / MedianPruner
The hyperparameter optimizer: Tree-structured Parzen Estimator sampling with median pruning, 200 trials, maximizing AUC-PR.
Purged walk-forward CV
Cross-validation in time order with purged folds, so each fold’s validation never overlaps its training labels.
AUC-PR
Area under the precision–recall curve — the tuning metric (robust to class imbalance).
XGBoost meta-label model
The gradient-boosted classifier that scores each setup’s probability of being a winner; embedded (base64) in the strategy file.
Risk-Box engine
The sequential trade simulator: all-in compounding, one position per asset, a symmetric target/safety “box”, and scheduled exits. The same engine scores Train acceptance and OOS.
Profit factor (PF)
Gross profit ÷ gross loss over OOS trades. The primary strategy objective (then min drawdown, then min time-in-market).
Selfcheck
The strategy file reloads its embedded model and reproduces its golden predictions — a built-in integrity check, not a security gate.
Dashboard / oos_metrics.db
The per-asset OOS results list. Each run (4.1) UPSERTs its verdict — end capital, return, PF, drawdown, win rate, trades, TIM, CV AUC-PR — into oos_metrics.db (a minimal, results-only SQLite store). make dashboard exports it to data/dashboard.json, which dashboard.html renders as a sortable table. The DB lives in Project/Structure/, outside the 7-file deliverable.
