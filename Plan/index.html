L1-L4 Data

Upstream Alpaca data is archived and exported to seed parquets. DuckDB becomes the shared input, then each asset writes clean 1h plus 1d/1w roll-ups.

L5-L6 Model Rows

Train/OOS split is index-based. Candidate bars use 1h momentum for side. Features are resolved by namespace and Y is an ATR Triple Barrier label.

L7-L9 Output

Optuna tunes XGB, Kelly is calibrated on Train out-of-fold growth, the model is embedded as base64, and every asset keeps the seven-file contract.

Run it from Project/Structure: make build-db && make run-asset TICKER=AAPL.
