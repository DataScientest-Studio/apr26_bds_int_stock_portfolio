S&P 500 per-asset ML pipeline

A minimal, self-contained trendline → triple-barrier → XGBoost/Optuna → OOS pipeline. Each ticker that passes the notebook leaves seven deliverable files.

🧭

Visualization

The data acquisition (1.3 Alpaca → QuantConnect/LEAN) → Layers 1.6 → 4.2 data flow and the 7 files each asset produces.
⚙️

Configurations

The parameters, split windows, and the XGBoost / Optuna search space the pipeline runs on.
📖

Glossary

The vocabulary: layers, Output-B, Risk-Box, triple-barrier, OOS, and friends.
📊

Dashboard

Per-asset OOS results from the 4.1 oos_metrics.db store: end capital, return, Profit Factor, drawdown, trades.
Run it: cd Project/Structure && make build-db && make run-asset TICKER=AAPL. The visualization shows only the layers the code implements — no SHA, no contracts, no extra QC gates.
